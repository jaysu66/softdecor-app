# 软装搭配AI场景生图 - 交付文档

## 一、项目概述

用户上传房间照片 → AI 分析空间 → 智能推荐商品 → AI 将商品合成到场景中 → 对话式微调 → 精修输出

**定位**：软装供应链公司的 AI 搭配生图工具，促进销售转化。

## 三、AI API 集成（核心）

### 3.1 Seedream 5.0（字节跳动 / 火山引擎 ARK）— 主力生图

| 项目 | 值 |
|------|---|
| **端点** | `https://ark.cn-beijing.volces.com/api/v3/images/generations` |
| **模型ID** | `doubao-seedream-5-0-260128` |
| **认证** | Bearer Token |
| **用途** | 场景合成（核心功能） |

**调用方式**：
- **多图合成模式**：`image_url`（房间底图，img2img 锚点）+ `image_urls`（房间 + 最多9件商品抠图），`strength=0.30`
- **单图回退模式**：仅 `image_url`（房间底图）+ 纯文本 prompt，`strength=0.40`
- **对话微调模式**：同多图模式，`strength=0.35`；同单图模式，`strength=0.45`
- **输出**：1920x1920 PNG，base64 编码

**降级策略**：多图失败 → 自动降级到单图模式

### 3.2 Seedream 3.0（火山引擎 ARK）— 精修

| 项目 | 值 |
|------|---|
| **端点** | `https://ark.cn-beijing.volces.com/api/v3/images/generations` |
| **模型ID** | `seedream-3.0` |
| **用途** | 合成图的真实感精修 |

**调用方式**：`image`（base64 原图）+ `prompt`（精修指令），`strength=0.35`，输出 1024x1024

### 3.3 Claude Sonnet 4.6（通过本地 Gateway）— 分析 & Prompt 生成

| 项目 | 值 |
|------|---|
| **端点** | `http://localhost:3000/v1/chat/completions` |
| **模型ID** | `claude-sonnet-4-6` |
| **认证** | Bearer `$GATEWAY_KEY` |

**三个用途**：

1. **房间分析**（`/api/analyze`）：上传房间图片 → Claude Vision 识别风格、房型、光线、色调、已有家具 → 输出 JSON
2. **合成 Prompt 生成**（`/api/compose`）：Claude 看房间图 + 商品信息 → 写 Seedream 多图引用 prompt（"第N张图"格式）
3. **Prompt 微调**（`/api/compose/adjust`）：根据用户反馈修改上一轮 prompt → 保持"第N张图"引用

### 3.4 Claude Opus 4.6（通过本地 Gateway）— 回退

| 项目 | 值 |
|------|---|
| **模型ID** | `claude-opus-4-6` |
| **用途1** | 精修回退：当 Seedream 3.0 不可用时，分析合成图给出评分和改进建议 |
| **用途2** | Agent 回退：当 Gemini tool-use 失败时，用 JSON 格式输出画布调整指令 |

### 3.5 Gemini 3.1 Pro（通过本地 Gateway）— 画布 Agent

| 项目 | 值 |
|------|---|
| **模型ID** | `gemini-3.1-pro-high` |
| **用途** | 自然语言操控画布上的商品位置/大小/旋转 |

**Tool-Use 工具**：

| 工具 | 功能 |
|------|------|
| `move_product(sku, x, y)` | 移动商品（百分比坐标 0-100） |
| `resize_product(sku, scale)` | 缩放（0.05-1.0） |
| `rotate_product(sku, angle)` | 旋转（0-360°） |
| `remove_product(sku)` | 从画布移除 |
| `add_product(sku)` | 从商品库添加到画布 |

### 3.6 未使用的依赖

`liblibai`（LibLib API SDK）在 `package.json` 中声明但**代码中未实际调用**。

## 五、业务流程（5步）

```
┌─────────────────────────────────────────────────────────────────┐
│  Step 1: 上传                                                    │
│  用户上传房间照片（自动压缩 1800px, JPEG 0.88）                      │
│  也可选择测试背景图                                                 │
├─────────────────────────────────────────────────────────────────┤
│  Step 2: AI 分析                                                 │
│  POST /api/analyze → Claude Sonnet Vision                        │
│  输出：房型、风格、光线、色调、已有家具、建议品类                        │
├─────────────────────────────────────────────────────────────────┤
│  Step 3: 智能推荐                                                 │
│  POST /api/recommend → 评分算法（风格30分+品类25分+空间15分+关联5分）  │
│  输出：方案A"精选搭配"(7件) + 方案B"混搭风格"(6件)                    │
│  用户可自由增删商品                                                 │
├─────────────────────────────────────────────────────────────────┤
│  Step 4: AI 场景合成 + 对话微调                                     │
│  POST /api/compose → Claude 写 Prompt → Seedream 5.0 多图合成      │
│  POST /api/compose/adjust → 用户文字反馈 → Claude 改 Prompt → 重新生成│
│  POST /api/agent-adjust → Gemini Tool-Use 调整画布商品位置           │
├─────────────────────────────────────────────────────────────────┤
│  Step 5: 精修 & 下载                                              │
│  POST /api/refine → Seedream 3.0 精修 / Claude Opus 反馈           │
│  用户下载最终图片                                                    │
└─────────────────────────────────────────────────────────────────┘
```
